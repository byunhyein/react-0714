# 바이브 코딩용 PRD - 세련된 회원 카드 리스트 출력 UI

## 1. 목적
- 다크 글래스모피즘(Glassmorphism) 스타일의 프리미엄 회원 카드 리스트 출력 UI를 구축합니다.
- 사용자가 직관적이고 시각적으로 즐거운 환경에서 팀원 또는 커뮤니티 멤버를 탐색하고 실시간으로 필터링할 수 있도록 합니다.

## 2. 배경
- 웹 서비스 내의 팀 소개나 멤버 목록 화면에서 단순한 리스트 형태를 탈피하여, 첫인상에서 사용자에게 시각적 감동(WOW 요소)을 주는 고품질 디자인이 필요합니다.
- 이에 현대적인 CSS 효과(반투명 블러, 네온 그라데이션, 미세한 호버 애니메이션 등)를 접목한 컴포넌트를 설계합니다.

## 3. 사용자 시나리오
- **방문자/관리자**는 메인 화면에 진입하자마자 세련되게 움직이는 다크 그라데이션 배경과 카드 목록을 마주합니다.
- **이름 검색:** 상단의 검색창에 찾고자 하는 멤버의 이름을 입력하면, 입력하는 즉시 해당하는 멤버만 필터링되어 화면에 표시됩니다.
- **역할 필터:** 검색창 밑의 역할 필터 칩(예: Developer, Designer, Manager 등)을 클릭해 특정 직무를 가진 팀원들만 실시간으로 걸러서 볼 수 있습니다.
- **마우스 인터랙션:** 개별 멤버 카드에 마우스를 올리면, 카드가 살짝 들어올려지며 네온 글로우 효과가 빛나 사용자 경험을 극대화합니다.

## 4. 핵심 기능
1. **멤버 리스트 그리드 렌더링**
   - 멤버 정보: 프로필 이미지, 이름, 직무(Role), 이메일, 활성 상태(Active/Inactive)
   - 반응형 그리드 구조로 디바이스 해상도에 맞춰 자동 정렬
2. **동적 이름 검색 기능**
   - 대소문자 구분 없이 실시간으로 이름을 필터링하여 목록 갱신
3. **역할(Role) 필터 칩**
   - 'All', 'Developer', 'Designer', 'Manager' 등 분류 기준 제공
   - 선택된 칩은 밝게 빛나며 시각적 피드백 제공
4. **멤버 상세 모달 (선택 기능)**
   - 멤버 카드를 클릭하면 더 상세한 소개 및 연락처 정보를 담은 모달 팝업 표시

## 5. 비기능 요구사항
- **디자인 & 테마:** 
  - 어두운 배경에 흐르는 컬러 그라데이션 오버레이 적용
  - 반투명 유리 느낌의 `backdrop-filter: blur(12px)` 및 미세한 흰색 테두리 적용 (글래스모피즘)
  - 상태 뱃지(Active/Inactive)는 부드러운 네온 컬러 발광 효과 적용
- **성능:** React 19 컴파일러를 활용하여 렌더링 성능 최적화, 이미지 로딩 최적화
- **반응형:** 데스크톱(3~4열), 태블릿(2열), 모바일(1열) 레이아웃 완벽 지원

## 6. 화면/컴포넌트 구조
- `src/App.jsx`: 전체 회원 데이터 상태 관리, 검색어 및 선택된 역할 필터 상태 관리, 메인 레이아웃 조합
- `src/components/SearchBar.jsx`: 세련된 다크 검색창 컴포넌트
- `src/components/FilterChips.jsx`: 역할별 필터 버튼 목록 컴포넌트
- `src/components/MemberGrid.jsx`: 멤버 카드 목록을 정렬하는 그리드 컨테이너
- `src/components/MemberCard.jsx`: 개별 회원의 상세 프로필 카드 컴포넌트
- `src/components/MemberModal.jsx`: 카드 클릭 시 오픈되는 상세 프로필 팝업
- `src/style.css`: 글래스모피즘 테마 및 반응형 그리드를 정의하는 메인 CSS 파일

## 7. 평가 기준
- **성공 기준:** 
  - 빌드 오류 없이 정상 실행되는가?
  - 이름 검색 및 필터 칩 동작 시 리스트가 부드럽게 실시간 반영되는가?
  - 모바일, 태블릿, 데스크톱 크기에서 디자인이 깨지지 않는가?
  - 다크 글래스모피즘 특유의 투명도와 그라데이션 글로우 효과가 제대로 구현되었는가?

## 8. 제약 조건
- CSS 프레임워크(TailwindCSS 등)는 사용하지 않고, 오직 **Vanilla CSS**로 커스텀 스타일링을 진행합니다.
- 이미지 플레이스홀더를 사용하는 대신 `generate_image`로 생성한 고품질 아바타 이미지를 적용합니다.

## 9. 작업 우선순위
- **필수:** 다크 글래스모피즘 스타일 적용, 검색 및 필터 칩 기능 개발, 멤버 카드 그리드 배치
- **선택:** 클릭 시 상세 모달 오픈, 멤버 활성 상태 토글 스위치

## 10. 참고 사항
- 아바타 이미지 에셋은 `public/images/` 디렉토리에 저장하여 정적 자산으로 참조할 예정입니다.
